# Java 依赖关系 (Relation) 提取约束规约

本文件定义了 `go-treesitter-dependency-analyzer` 在 Java 语言中识别和提取的各种依赖关系类型及其元数据。

## 1. 关系概览

关系主要分为三大类：**层级关系 (Hierarchy)**、**结构关系 (Structural)** 和 **动作关系 (Action)**。

| 关系类型 | 描述 | 提取来源 |
| :--- | :--- | :--- |
| **Import** | 文件导入关系 | `import_declaration` |
| **Contain** | 父子容器关系 | 物理嵌套结构 (类包含方法等) |
| **Extend** | 类或接口的继承 | `extends` 关键字 |
| **Implement** | 类实现接口 | `implements` 关键字 |
| **Annotation** | 注解引用 | 元素上的 `@Annotation` |
| **Parameter** | 方法参数类型依赖 | 方法签名的参数部分 |
| **Return** | 方法返回值类型依赖 | 方法签名的返回类型部分 |
| **Throw** | 异常抛出依赖 | `throws` 签名或 `throw` 语句 |
| **TypeArg** | 泛型参数依赖 | 泛型符号 `<T>` 中的类型 |
| **Call** | 方法/构造函数调用 | 方法调用、new、super()、方法引用 |
| **Create** | 对象/数组创建 | `new` 表达式 |
| **Assign** | 变量/字段赋值 | 赋值语句、初始化、自增减 |
| **Use** | 变量/字段读取 | 表达式中的变量引用 |
| **Cast** | 类型转换/判定 | 强制转换 `(Type)` 或 `instanceof` |
| **Capture** | 闭包变量捕获 | Lambda/匿名类引用外部作用域变量 |

## 2. 关系元数据 (Metadata/Mores) 详情

每个关系都会携带特定的增强元数据，用于后续的架构分析和过滤。

### 2.1 Call (调用)
- `java.call.is_static`: 是否为静态调用。
- `java.call.is_constructor`: 是否为构造函数调用。
- `java.call.is_chained`: 是否为链式调用 (如 `a().b()`)。
- `java.call.is_functional`: 是否为方法引用 (如 `System.out::println`)。
- `java.call.receiver`: 调用接收者文本 (如 `obj`, `this`, `super`)。
- `java.call.receiver_type`: 接收者的类型 QN。
- `java.call.enclosing_method`: 所在的顶层方法 QN (跳过 Lambda/块级作用域)。

### 2.2 Create (创建)
- `java.create.variable_name`: 接收该对象的变量名。
- `java.create.is_array`: 是否为数组创建。

### 2.3 Assign & Use (赋值与使用)
- `java.assign.operator`: 赋值操作符 (`=`, `+=`, `++` 等)。
- `java.assign.is_initializer`: 是否为变量定义时的初始化。
- `java.assign.value_expression`: 赋值右值的原始文本。
- `java.use.receiver`: 访问的前缀 (如 `this.field` 中的 `this`)。
- `java.rel.is_capture`: 是否发生了跨作用域捕获。

### 2.4 Structural (结构)
- **Parameter**: 包含 `index` (参数位置)、`name` (参数名)、`is_varargs` (是否为变长参数)。
- **Return**: 包含 `is_primitive` (是否为基础类型)、`is_array` (是否为数组)。
- **Annotation**: 包含 `target` (注解作用的目标类型，如 `METHOD`, `FIELD`)。
- **TypeArg**: 包含 `index` (泛型参数索引)。

### 2.5 Throw (抛出)
- `java.throw.is_signature`: 是否来自方法签名的 `throws`。
- `java.throw.is_runtime`: 是否为显式 `new` 出来的运行时异常抛出。
- `java.throw.is_rethrow`: 是否为异常重抛 (抛出捕获的变量)。

## 3. 提取逻辑约束

- **泛型擦除**: 在建立关系目标 (Target) 时，会移除泛型信息以匹配定义的 QN (例如 `List<String>` 建立到 `java.util.List` 的依赖)。
- **基础类型过滤**: 默认不建立指向 `int`, `char` 等基础类型的依赖关系。
- **隐式 this**: 当访问当前类的字段或方法无前缀时，`receiver` 自动填充为 `this`。
- **Capture 自动发现**: 如果一个 `Use` 或 `Assign` 关系的 `Source` 位于 Lambda 或匿名类内部，且其 `Target` 定义在外部方法中，系统会自动额外生成一条 `Capture` 关系。
